How it works

Deterministic rules, not AI

Every URL in the output came from your input. No pages are invented, guessed, or hallucinated. The tool applies a fixed set of rules to produce a clean, valid sitemap.xml.

Step 1 — Parse and validate

Each line is trimmed and tested against a URL pattern. Lines that are not valid absolute URLs (missing scheme or domain) are flagged and skipped. Relative paths like /blog are left out unless a base domain is detected from the first URL.

Step 2 — Deduplicate

Duplicate URLs (including trailing-slash variants) are removed. The first occurrence is kept. Canonicalization removes fragments (#section) since search engines ignore them.

Step 3 — Assign priority and changefreq

Priority is assigned by URL depth: homepage = 1.0, one level deep = 0.8, two levels = 0.7, three or more = 0.6. Changefreq follows the same logic unless you override it in the options: homepage = weekly, everything else = monthly.

Step 4 — Build the XML

The tool wraps each URL in a standard <url> element with <loc>, <lastmod>, <changefreq>, and <priority>. The output is indented for readability and validates against the sitemaps.org schema.

SEO best practices

What makes a good sitemap

Only include indexable URLs

Exclude pages with noindex meta tags, canonical redirects, 4xx errors, and URL parameters that create duplicate content. A smaller, clean sitemap performs better than a large one with junk URLs.

Keep lastmod accurate

Google uses lastmod as a signal for recrawl priority. Only update it when the page content actually changes — setting it to today for every page on every deploy teaches Google to ignore it.

XML is for search engines, HTML is for users

The XML sitemap tells crawlers which URLs exist. The HTML sitemap helps human visitors navigate a large site. Both are useful, but they serve different audiences.

Reference the sitemap in robots.txt

Add Sitemap: https://example.com/sitemap.xml as the last line of your robots.txt file. This tells all compliant crawlers where to find it without waiting for them to discover it via Search Console.

Use sitemap index files for large sites

A single sitemap supports up to 50,000 URLs and 50 MB. For larger sites, create multiple sitemaps and reference them in a sitemap index file.

Submit to Google Search Console

Upload your sitemap.xml to your website root, then go to Google Search Console, select your property, click Sitemaps in the sidebar, and submit the full URL. Re-submit any time the sitemap changes significantly.
