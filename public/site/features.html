What does ConversionCRM do?

ConversionCRM is free trial conversion software for SaaS. Drop in one script tag and it auto-tracks behavior, scores every user 0–100 across six layers, assigns a lifecycle stage, and sends 8 behavior-triggered emails — from welcome through win-back — on autopilot. Setup takes about 3 minutes. Free plan included.

Feature 01

Zero-config behavior tracking

Paste one script tag before </body>. Page views, clicks, time on page, and SPA route changes are tracked instantly — no event schema to design, no analyst to loop in.

Auto-tracked events

Page views, clicks, scroll depth, time on page, and SPA navigation are captured automatically. Zero extra code for the basics.

Custom events with one line

ccrm.track("feature_used") attaches any domain event to the same user timeline. Pricing visits, feature completions, limit hits.

Named users from identify()

Call ccrm.identify(userId, { email }) after login and every event has a name on it — so the dashboard is people, not anonymous IDs.

REST API for backend events

Server-side events (payment started, export completed) arrive via a simple REST call using your workspace API key. Same timeline, same score.

Feature 02

6-layer engagement scoring, 0–100

A single number that means something. Recomputed every night from the last 7 days of behavior — and live every 3 seconds while the dashboard is open.

Recency (20 pts)

How recently the user was active. A session today scores 20; no activity in 7 days scores 0. Decay is linear over the window.

Frequency (20 pts)

Session count over the last 7 days. Casual dippers score low; power users who open the app daily score full.

Depth (15 pts)

Unique pages and features visited. Surface-level visitors and deep explorers separate immediately.

Key feature hit (20 pts)

Did they trigger your configured aha-moment event? This layer alone predicts retention better than the other five combined.

Time spent (10 pts)

Cumulative active time in the last 7 days. Passive tab-openers get filtered out; engaged users get credit.

Buying intent (15 pts)

Pricing page visits, upgrade button clicks, and usage-limit hits. When this layer spikes, the upgrade email goes out that night.

Feature 03

Automatic lifecycle stages

Seven stages, assigned from the score and time signals — no rules to write, no segments to maintain. The dashboard becomes your pipeline at a glance.

Signup

New user, first session within 24 hours. The welcome email fires immediately at this stage.

Onboarding

Active in the first 7 days but aha moment not yet hit. Feature nudge email targets this stage.

Active

Returning user, score 30–70. Check-in email runs if engagement starts declining.

Conversion ready

Score 71–100 plus buying-intent signals. This is your PQL list. Upgrade offer and urgency emails fire here.

Going quiet

Score dropped below 20. Churn prevention email triggers before the user is fully gone.

Churned & Paid

30+ days inactive (Churned) or confirmed payment (Paid). Paid users are permanently excluded from lifecycle email.

Feature 04

8 behavior-triggered lifecycle emails

Plain, founder-style emails under your sender name. One email per user per night maximum, with cooldowns on every trigger. Paying users never receive them.

01

Welcome

Fires the moment a user signs up. Sets tone, confirms tracking is live, invites them to the aha moment.

02

Feature nudge

User is in Onboarding but hasn't hit the key feature. Sent 24 hours in if no aha-moment event is recorded.

03

Value demo

Shares a concrete example of what the product does for someone in their stage. Hits when score is climbing.

04

Check-in

For Active users whose score has plateaued for 5+ days. A short, personal note that prompts a reply.

05

Upgrade offer

Score crosses 71. This is the moment they're most ready — the email matches the intent.

06

Urgency

User visited pricing or clicked an upgrade button. Follows up the same night with a direct offer.

07

Churn prevention

Score dropped from Active to Going quiet. Sent before the user disappears to bring them back.

08

Limit upgrade

User hit a real usage limit. Explains the constraint and presents the upgrade path clearly.

Feature 05

Real-time user dashboard

Every user, their score, their stage, and their full activity feed — updating every 3 seconds while you watch. No BI tool, no SQL.

Live score breakdown

Click any user and see all 6 score layers with their individual points. You know exactly why someone is at 67 vs 82.

Activity feed per user

Every tracked page, click, and custom event in chronological order. See the moment they hit pricing, the feature they keep returning to, and when they went quiet.

Stage pipeline view

Filter by stage to see all users in each bucket. Conversion-ready is your PQL list — work it directly from the dashboard with the manual email composer.

Manual email composer

Send a personal note to any user directly from their profile. Uses the same email transport as the automated system — no separate tool.

Feature 06

Dual-path email delivery

Every email goes out under your sender name. Choose our infrastructure or bring your own SMTP — switching takes 30 seconds in Settings.

Resend (default)

Delivered via ConversionCRM's verified sending domain at mail.conversioncrm.co. Ready in 3 minutes, no DNS records to add on your side.

Your own SMTP

Enter your Gmail, Outlook, Amazon SES, or any SMTP credentials. Emails send from your actual domain. Replies land in your inbox.

Sender name + reply-to

Every email shows your name and your reply-to address. Automated emails feel like they came from a person — because the reply conversation does.

Guardrails built in

One lifecycle email per user per nightly batch. Per-trigger cooldowns. Paid users permanently excluded. No spam, by design.
